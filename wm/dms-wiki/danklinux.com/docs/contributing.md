## How to Contribute[â€‹](#how-to-contribute "Direct link to How to Contribute")

### Reporting Issues[â€‹](#reporting-issues "Direct link to Reporting Issues")

Found a bug or have a feature request? Please open an issue on the relevant repository:

* [DankMaterialShell Issues](https://github.com/AvengeMedia/DankMaterialShell/issues)
* [dgop Issues](https://github.com/AvengeMedia/dgop/issues)
* [dsearch Issues](https://github.com/AvengeMedia/danksearch/issues)

### Pull Requests[â€‹](#pull-requests "Direct link to Pull Requests")

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages: `git commit -m "<component/area>: new feature description"`
6. Push to your fork: `git push origin feature/my-feature`
7. Open a Pull Request

### Code Style[â€‹](#code-style "Direct link to Code Style")

* **Go**: Follow standard Go formatting (`go fmt`, `make fmt`)
  + Include appropriate test coverage
  + Fix lint errors `golangci-lint run`
* **TypeScript/JavaScript**: Use consistent formatting
* **QML**: Format changed files with `qmlfmt -t 4 -i 4 -b 250 -w MY_FILE.qml`
  + Requires [qmlfmt](https://github.com/jesperhh/qmlfmt) tool
* **Documentation**: Use clear, concise languag
